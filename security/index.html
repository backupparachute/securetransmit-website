Security

Security is at the core of everything we do. Learn how Secure Transmit protects your data and ensures compliance.

Encryption

All data is encrypted in transit and at rest using industry-standard AES-256 encryption. We use secure protocols (SFTP, FTPS, HTTPS) for all file transfers.

Security modes

SecureTransmit offers tiered security modes so you choose the tradeoff that fits the workflow:

  • Standard mode — AES-256 at rest, TLS in transit, least-privilege access, full audit trail. The default for workflows that need transforms.
  • Sealed mode — pre-encrypted payloads that SecureTransmit moves and logs but never decrypts.
  • Paranoid mode — delete-on-ack. The moment the destination acknowledges receipt, the file is gone. SHA-256 hash and audit record stay forever for non-repudiation. A breach exposes only in-flight data.
  • Island mode — outbound-only architecture. Partners push to SecureTransmit, your systems pull from SecureTransmit, and you close every inbound B2B port on your own firewall.

Compliance posture

SecureTransmit is engineered so you can build compliant workflows on top of it — immutable audit trails, hash-verified records, configurable retention, and least-privilege access. We are not advertising specific certifications until we’ve earned them. Ask us where we are in the process.

User Safety

  • Multi-factor authentication (MFA) for all accounts
  • Role-based access controls
  • Continuous monitoring and threat detection
  • Regular security audits and penetration testing
